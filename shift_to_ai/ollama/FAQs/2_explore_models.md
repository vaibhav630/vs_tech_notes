# which site to visit to check these models

To explore and decide which models to use with Ollama, there are a few **reliable websites** you should bookmark 👇

---

# 🌐 1. Official Ollama Model Library (BEST place to start)

👉 Visit:
**[https://ollama.com/library](https://ollama.com/library)**

### Why this is the best:

* Shows all models you can directly run with `ollama pull`
* Gives exact commands like:

  ```bash
  ollama run llama3
  ```
* Lists:

  * Model sizes
  * Variants (8B, 70B, etc.)
  * Descriptions

### What you’ll find there:

* Llama 3
* Mistral
* Mixtral
* Code Llama

👉 This is your **primary site** — 90% of your decisions can be made here.

---

# 🧠 2. Hugging Face (for deeper exploration)

👉 Visit:
**[https://huggingface.co/models](https://huggingface.co/models)**

### Why use this:

* Massive collection of models (not all work directly in Ollama)
* Detailed benchmarks, community reviews
* Advanced filtering (size, task, language, etc.)

### When to use:

* You want to compare models deeply
* You’re exploring beyond Ollama defaults
* You want niche models (finance, medical, etc.)

---

# 📊 3. LLM Leaderboards (for performance comparison)

👉 Visit:

* **[https://huggingface.co/spaces/lmsys/chatbot-arena-leaderboard](https://huggingface.co/spaces/lmsys/chatbot-arena-leaderboard)**

### Why useful:

* Shows which models are *actually performing best*
* Based on real user voting

👉 Helps answer:
“Which model is smarter?”

---

# 🧪 4. Reddit / Community (real-world feedback)

Search:

* “best ollama models reddit”

👉 You’ll find practical insights like:

* Which model is fastest on Mac
* Which works best with tools like n8n

---

# 🧠 Simple workflow (what YOU should do)

1. Go to **Ollama Library** ✅
2. Pick:

   * llama3 → general use
   * mistral → faster
3. Pull and test locally
4. If curious → explore Hugging Face

---

# ⚡ Pro tip (saves time)

Don’t over-research initially.

👉 Just start with:

```bash
ollama run llama3
```

Then compare with:

```bash
ollama run mistral
```

You’ll learn faster by **trying than reading**.
